# Перемычки

Модуль автоматизации оформления перемычек для спецификаций.

Сейчас модуль предоставляет:

- кнопку во вкладке `КР`;
- read-only диагностику текущего выделения или активного вида;
- группировку найденных экземпляров по Revit `TypeId`;
- рекурсивную проверку доступных через `GetSubComponentIds()` вложенных компонентов;
- проверку наличия модельной геометрии и отчёт о причинах исключения;
- modeless-окно выбора готовых типоразмеров;
- read-only preview будущих имён сборки, бокового вида и PNG-изображения;
- обновление данных после изменения выделения через безопасный Revit `ExternalEvent`;
- read-only preflight выбранных составов через `AssemblyInstance.AreElementsValidForAssembly()` и `IsValidNamingCategory()`;
- обнаружение уже существующей сборки с детерминированным TrueBIM-именем до повторного запуска;
- автоматический preflight по нажатию основной кнопки, чтобы выбранный готовый типоразмер не упирался в неочевидный disabled-state;
- создание ровно одной подтверждённой Assembly после отдельного согласия пользователя;
- атомарное создание и именование в двух транзакциях внутри общего `TransactionGroup`;
- создание одного бокового assembly view слева в масштабе `1:10`, с высоким уровнем детализации, стилем скрытых линий и защитой от дубликатов.

Явно выделенные `FamilyInstance` принимаются как кандидаты без проверки имени. Автопоиск на активном виде временно использует фрагменты `перемыч` и `lintel` в имени семейства, типоразмера или экземпляра. Это правило должно быть уточнено после проверки рабочего RVT-файла.

Имена формируются детерминированно с префиксом `TB_Перемычка_`, очищаются от недопустимых символов и содержат Revit `TypeId` для различения совпадающих названий. Кнопка создания активируется при выборе ровно одного готового типоразмера, сама выполняет preflight через Revit API и только затем показывает подтверждение записи. Отдельная кнопка `Проверить без создания` остаётся для диагностики одного или нескольких составов.

Диагностика, preview и preflight не открывают транзакцию и не изменяют модель Revit. Операции записи разделены: сначала создаётся одна Assembly из вложенных компонентов, затем отдельной кнопкой — один боковой вид `1:10`. Аннотации и изображение пока не создаются. Полная последовательность, критерии готовности и открытые вопросы находятся в [`docs/Lintels/ROADMAP.md`](../../../../../../docs/Lintels/ROADMAP.md).

Основной поток проверен 15.07.2026 в Revit 2023 на отсоединённой тестовой копии предоставленного RVT: одна Assembly создана, повторный запуск распознал её без дублирования, затем создан один боковой вид `ElevationLeft` в масштабе `1:10`.
