# Перемычки

Модуль автоматизации оформления перемычек для спецификаций.

Сейчас модуль предоставляет:

- кнопку во вкладке `КР`;
- read-only диагностику текущего выделения или активного вида;
- группировку найденных экземпляров по Revit `TypeId`;
- рекурсивную проверку доступных через `GetSubComponentIds()` вложенных компонентов;
- проверку наличия модельной геометрии и отчёт о причинах исключения;
- модальный первый шаг мастера `Откуда взять перемычки` с режимами `Текущее выделение`, `Активный вид`, `Весь проект` и `Результаты, созданные TrueBIM`;
- рабочий переход из модального шага в окно пакетного выбора для режимов `Текущее выделение`, `Активный вид` и `Результаты, созданные TrueBIM`;
- явное объяснение шагов 2–4: выбор типоразмеров, создание сборок, выбор семейства рамки и создание видов;
- read-only preview будущих имён сборки, бокового вида и PNG-изображения;
- обновление данных после изменения выделения через безопасный Revit `ExternalEvent`;
- read-only preflight выбранных составов через `AssemblyInstance.AreElementsValidForAssembly()` и `IsValidNamingCategory()`;
- обнаружение уже существующей сборки с детерминированным TrueBIM-именем до повторного запуска;
- автоматический preflight всех отмеченных типоразмеров по нажатию основной кнопки;
- пакетное создание одной Assembly для каждого отмеченного типоразмера после общего подтверждения;
- атомарное создание и именование в двух транзакциях внутри общего `TransactionGroup`;
- пакетное создание боковых assembly view слева в масштабе `1:10`, с высоким уровнем детализации, стилем скрытых линий и защитой от дубликатов;
- повторное оформление уже существующего TrueBIM-вида без создания нового вида;
- один габаритный линейный размер по крайним граням геометрии и Spot Elevation с префиксом `отм.` без выноски по нижней грани перемычек;
- автоматическое подключение `Y:\01 - REVIT\03 - FAMILIES 2017\Аннотации\•• (Аннотация) Рамка аннотации.rfa`, когда библиотека доступна, и явный выбор другого `.rfa` как резерв; TrueBIM загружает семейство и размещает один экземпляр по центру каждого вида вместо построения рамки detail lines;
- центрирование crop по геометрии с временным минимальным форматом `1050 × 385 мм` в модели;
- экспорт каждого оформленного вида в PNG шириной `1600 px` в локальный каталог `%LOCALAPPDATA%\TrueBIM\Lintels\<проект>`;
- импорт или обновление PNG как `ImageType` без дублирования и назначение исходному типу перемычки через встроенный параметр `Изображение типоразмера`;
- безопасный повторный запуск: модуль пересоздаёт только собственные аннотации, записанные в Extensible Storage;
- доступ к повторному оформлению непосредственно с уже открытого бокового вида TrueBIM Assembly.

Явно выделенные `FamilyInstance` принимаются как кандидаты без проверки имени. Автопоиск на активном виде временно использует фрагменты `перемыч` и `lintel` в имени семейства, типоразмера или экземпляра. Это правило должно быть уточнено после проверки рабочего RVT-файла.

Имена формируются детерминированно с префиксом `TB_Перемычка_`, очищаются от недопустимых символов и содержат Revit `TypeId` для различения совпадающих названий. Кнопка шага 3 активируется при выборе одного или нескольких готовых типоразмеров, сама выполняет preflight через Revit API и затем показывает единое подтверждение записи. Отдельная кнопка `Проверить выбранные` позволяет проверить пакет без изменения модели. Источник `Результаты, созданные TrueBIM` означает только те исходные типоразмеры, для которых в текущем проекте найдена Assembly с детерминированным именем TrueBIM; пока таких сборок нет, источник честно показывает состояние `Пока пусто`.

Диагностика, preview и безопасная проверка не открывают транзакцию и не изменяют модель Revit. Первый шаг мастера является отдельным модальным окном; шаги 2–4 выполняются в одном рабочем окне и постоянно видны в поясняющей строке. Операции записи разделены: сначала для отмеченных строк создаются Assembly, затем пользователь выбирает одно семейство рамки `.rfa`, после чего отдельной кнопкой создаются или переиспользуются боковые виды `1:10`, применяется оформление, экспортируется PNG и назначается `Изображение типоразмера`. Выбор строк сохраняется после обновления данных из Revit. Ошибка одного типоразмера не останавливает уже обработанные строки; результат показывается отдельно по каждой сборке и виду. Марки компонентов пока не создаются. Полная последовательность и критерии готовности находятся в [`docs/Lintels/ROADMAP.md`](../../../../../../docs/Lintels/ROADMAP.md).

Основной одиночный поток проверен 15.07.2026 в Revit 2023 на отсоединённой тестовой копии предоставленного RVT. После локального обновления payload созданы Assembly `3482693` и боковой вид `3482696`; лог подтвердил `Dimension=True`, `Elevation=True`, `Frame=True` и `CropAdjusted=True`. Новые пакетный сценарий и размещение явно выбранного семейства рамки добавлены 23.07.2026 и требуют ручной визуальной проверки в Revit на рабочей копии RVT и файле `•• (Аннотация) Рамка аннотации.rfa`.
