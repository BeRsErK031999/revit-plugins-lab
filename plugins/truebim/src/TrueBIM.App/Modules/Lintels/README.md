# Перемычки

Модуль автоматизации оформления перемычек для спецификаций.

Сейчас модуль предоставляет:

- кнопку во вкладке `КР`;
- read-only диагностику текущего выделения или активного вида;
- группировку найденных экземпляров по Revit `TypeId`;
- рекурсивную проверку доступных через `GetSubComponentIds()` вложенных компонентов;
- проверку наличия модельной геометрии и отчёт о причинах исключения;
- modeless-окно выбора готовых типоразмеров;
- read-only preview будущих имён сборки, бокового вида и PNG-изображения;
- обновление данных после изменения выделения через безопасный Revit `ExternalEvent`;
- read-only preflight выбранных составов через `AssemblyInstance.AreElementsValidForAssembly()` и `IsValidNamingCategory()`;
- обнаружение уже существующей сборки с детерминированным TrueBIM-именем до повторного запуска.

Явно выделенные `FamilyInstance` принимаются как кандидаты без проверки имени. Автопоиск на активном виде временно использует фрагменты `перемыч` и `lintel` в имени семейства, типоразмера или экземпляра. Это правило должно быть уточнено после проверки рабочего RVT-файла.

Имена формируются детерминированно с префиксом `TB_Перемычка_`, очищаются от недопустимых символов и содержат Revit `TypeId` для различения совпадающих названий. Кнопка создания сборок намеренно отключена с видимым объяснением.

Диагностика, preview и preflight не открывают транзакцию и не изменяют модель Revit. Кнопка `Создать сборки` остаётся заблокированной до проверки рабочего RVT-файла. Полная последовательность, критерии готовности и открытые вопросы находятся в [`docs/Lintels/ROADMAP.md`](../../../../../../docs/Lintels/ROADMAP.md).
