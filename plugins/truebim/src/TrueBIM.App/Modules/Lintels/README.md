# Перемычки

Модуль автоматизации оформления перемычек для спецификаций.

Сейчас модуль предоставляет:

- кнопку во вкладке `КР`;
- read-only диагностику текущего выделения или активного вида;
- группировку найденных экземпляров по Revit `TypeId`;
- рекурсивную проверку доступных через `GetSubComponentIds()` вложенных компонентов;
- проверку наличия модельной геометрии и отчёт о причинах исключения;
- модальный первый шаг мастера `Откуда взять перемычки` с режимами `Текущее выделение`, `Активный вид`, `Весь проект` и `Уже созданные`;
- рабочий переход из модального шага в существующее окно выбора для режимов `Текущее выделение` и `Активный вид`;
- read-only preview будущих имён сборки, бокового вида и PNG-изображения;
- обновление данных после изменения выделения через безопасный Revit `ExternalEvent`;
- read-only preflight выбранных составов через `AssemblyInstance.AreElementsValidForAssembly()` и `IsValidNamingCategory()`;
- обнаружение уже существующей сборки с детерминированным TrueBIM-именем до повторного запуска;
- автоматический preflight по нажатию основной кнопки, чтобы выбранный готовый типоразмер не упирался в неочевидный disabled-state;
- создание ровно одной подтверждённой Assembly после отдельного согласия пользователя;
- атомарное создание и именование в двух транзакциях внутри общего `TransactionGroup`;
- создание одного бокового assembly view слева в масштабе `1:10`, с высоким уровнем детализации, стилем скрытых линий и защитой от дубликатов;
- повторное оформление уже существующего TrueBIM-вида без создания нового вида;
- один габаритный линейный размер по крайним граням геометрии и высотную отметку по верхней грани;
- детерминированное построение ограничивающей рамки четырьмя detail lines: системный стиль `Невидимые линии`, а при его отсутствии — стандартный стиль detail line текущего проекта;
- центрирование crop по геометрии с временным минимальным форматом `1050 × 385 мм` в модели;
- безопасный повторный запуск: модуль пересоздаёт только собственные аннотации, записанные в Extensible Storage;
- доступ к повторному оформлению непосредственно с уже открытого бокового вида TrueBIM Assembly.

Явно выделенные `FamilyInstance` принимаются как кандидаты без проверки имени. Автопоиск на активном виде временно использует фрагменты `перемыч` и `lintel` в имени семейства, типоразмера или экземпляра. Это правило должно быть уточнено после проверки рабочего RVT-файла.

Имена формируются детерминированно с префиксом `TB_Перемычка_`, очищаются от недопустимых символов и содержат Revit `TypeId` для различения совпадающих названий. Кнопка создания активируется при выборе ровно одного готового типоразмера, сама выполняет preflight через Revit API и только затем показывает подтверждение записи. Отдельная кнопка `Проверить без создания` остаётся для диагностики одного или нескольких составов.

Диагностика, preview и безопасная проверка не открывают транзакцию и не изменяют модель Revit. Первый шаг нового мастера уже является модальным; существующее рабочее окно временно остаётся следующим переходным шагом, пока остальные страницы мастера переносятся по RoadMap v2. Операции записи разделены: сначала создаётся одна Assembly из вложенных компонентов, затем отдельной кнопкой создаётся или переиспользуется один боковой вид `1:10` и применяется базовое оформление. Ошибка отдельной аннотации не откатывает успешно созданный вид и остальные элементы оформления, а причина выводится пользователю и записывается в лог. Марки компонентов и изображение пока не создаются. Полная последовательность и критерии готовности находятся в [`docs/Lintels/ROADMAP.md`](../../../../../../docs/Lintels/ROADMAP.md).

Основной поток проверен 15.07.2026 в Revit 2023 на отсоединённой тестовой копии предоставленного RVT. После локального обновления payload созданы Assembly `3482693` и боковой вид `3482696`; лог подтвердил `Dimension=True`, `Elevation=True`, `Frame=True` и `CropAdjusted=True`. Визуальная проверка выявила, что прежний поиск семейства по подстроке `Рамка` вставлял поверх геометрии табличную аннотацию с незаполненными метками `?`. Неоднозначный ресурс заменён собственным невидимым ограничивающим контуром, а выноска высотной отметки направлена внутрь области. Исправление требует повторной проверки после обновления payload и перезапуска Revit 2023.
