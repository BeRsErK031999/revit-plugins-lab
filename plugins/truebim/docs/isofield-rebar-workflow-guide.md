# Методичка: армирование по изополям

![Армирование по изополям: порядок работы](assets/isofield-rebar-workflow-card.svg)

Эта методичка относится только к текущему модулю `Армирование по изополям`. Она описывает текущий безопасный режим: входной JSON или изображение, preview, выбор host-элемента, расчёт правил и контролируемое создание пробного армирования.

## Где открыть справку в окне

![Окно модуля с кнопкой методички](assets/isofield-rebar-window-guide.svg)

В правом верхнем углу окна есть кнопка `?`. При наведении она показывает короткую подсказку, а по клику открывает встроенную методичку с картинками и примером. Эта справка не относится к другим инструментам TrueBIM.

## Как работает текущий режим

1. Пользователь выбирает исходный файл изополей: JSON fixture или изображение.
2. JSON читается сразу, а изображение передается в настроенный CLI-worker; если worker не настроен, используется безопасная заглушка.
3. Результат распознавания превращается в контуры и показывается в окне модуля.
4. По отдельной кнопке контуры можно показать в Revit как управляемые preview-линии.
5. Пользователь выбирает host: простую прямую стену или горизонтальную плиту.
6. Для плиты пользователь задаёт две узнаваемые точки на карте и указывает соответствующие точки на верхней грани. Плагин показывает read-only overlay контура, отверстий и зон.
7. Привязка с точками или зонами вне допустимого контура блокирует дальнейший расчёт; для стены этот шаг не требуется.
8. Правила армирования рассчитываются в read-only режиме и показывают диагностику.
9. Пробное армирование создаётся только по кнопке `Создать пробное армирование` и после явного подтверждения.

До шага создания пробного армирования модуль не должен записывать арматуру в модель Revit.

## Пример

![Пример прохождения JSON-контура до пробного армирования](assets/isofield-rebar-example-flow.svg)

Для быстрой проверки используйте файл `docs/IsoFieldRebar/examples/sample-wall-zones.json`.

1. Откройте вкладку `TrueBIM`.
2. На панели `БИМ` нажмите `Армирование по изополям`.
3. Нажмите `Выбрать файл` и укажите `sample-wall-zones.json`.
4. Проверьте, что в блоке `Предпросмотр контуров` появились линии изополей.
5. При необходимости нажмите `Показать в Revit`, чтобы увидеть временные линии на активном 2D-виде.
6. Нажмите `Выбрать стену/плиту` и выберите простую стену. Для проверки плиты используйте `sample-slab-zones.json`, укажите две пары соответствующих точек и нажмите `Проверить привязку`.
7. Для плиты проверьте overlay: внешний контур зелёный, отверстия показаны пунктиром, обе контрольные точки пронумерованы. Красный контур означает блокирующую ошибку.
8. Нажмите `Рассчитать правила` и проверьте список правил и диагностику.
9. Нажмите `Создать пробное армирование` только если правила валидны и вы готовы изменить модель.
10. После проверки созданного пробного элемента выполните Revit Undo.

## Что считается корректным результатом

- JSON `schemaVersion: "1.0"` читается без ошибки.
- Контуры видны в окне модуля.
- Preview в Revit создается только как временные управляемые линии и очищается через `Очистить`.
- Для горизонтальной плиты двухточечная привязка показывает внешний контур, отверстия и зоны; несовместимая привязка блокирует правила.
- Правила армирования показываются до записи в модель.
- Тестовая арматура создается только после подтверждения.
- Один Revit Undo возвращает модель в исходное состояние после пробной записи.

## Ограничения текущего режима

- Реальное распознавание контуров остается внешним worker-ом.
- Запись в модель сейчас создаёт пробные стержни, а не финальную раскладку.
- Стены поддерживаются в простом прямом сценарии.
- Overlay читает верхнюю грань простой горизонтальной плиты и учитывает отверстия только при read-only проверке. Зоны пока не обрезаются, а пробные стержни по-прежнему размещаются упрощённо по bounding box.
- Справка описывает только модуль `Армирование по изополям`.

## Что приложить к отчету об ошибке

- версию Revit;
- входной JSON или изображение;
- тип host-элемента: стена или плита;
- скриншот окна `Армирование по изополям`;
- лог `%APPDATA%\TrueBIM\Logs\truebim.log`;
- описание шага: загрузка файла, preview, выбор host, расчёт правил или пробная запись.

После сборки установщика этот файл должен лежать в установленном пакете по пути `%APPDATA%\TrueBIM\<RevitVersion>\Docs\isofield-rebar-workflow-guide.md`, а картинки должны лежать рядом в `%APPDATA%\TrueBIM\<RevitVersion>\Docs\assets`.
