# План работ по BIM-инструментам

## Исходный контекст

Первый BIM-набор в проекте уже реализован внутри существующей Revit-вкладки `TrueBIM`. Он закрывает задачи проверки модели, параметров и администрирования:

- `Цвета по параметрам`
- `Копирование параметров`
- `Рабочие наборы`
- `ParaManager`

Новое ТЗ добавляет отдельный пакет `БИМ-оформление`: размеры, марки, штампы, виды, выпуск, библиотека семейств и координация. Чтобы не переносить существующие рабочие команды, текущий план сохраняет хост-вкладку `TrueBIM` и добавляет новые BIM-панели внутри неё.

## Пакет БИМ-Оформление

Статус: каркас добавлен, рабочие MVP `Экспорт PDF/DWG`, `Диспетчер семейств`, `Оформить штамп` и `Автомарки` реализованы. Остальные новые команды открывают WPF-окно-заглушку с кнопками `Предпросмотр`, `Выполнить`, `Закрыть`, сохраняют служебный JSON в `%AppData%/TrueBIM/BimTools/<tool>/settings.json` и пока не изменяют модель Revit.

## Задача 0.1: Экспорт PDF/DWG

Статус: готово.

Цель: дать первый полезный инструмент пакета `БИМ-оформление` без дублирования низкоуровневого Revit API-кода из модуля `Печать`.

Фактически реализовано:

- команда `BatchExportCommand` запускается из панели `Выпуск`;
- окно `BatchExportWindow` показывает листы активного документа;
- есть фильтр по номеру/имени листа, выбор всех/снятие выбора и отдельные чекбоксы `PDF`/`DWG`;
- правило имени по умолчанию: `{SheetNumber}_{SheetName}_{Revision}`;
- поддержаны токены `{SheetNumber}`, `{SheetName}`, `{Revision}`, `{Date:yyyy-MM-dd}`, `{Counter:000}`, `{SheetParameter:...}`, `{ProjectParameter:...}`;
- preview имен очищает недопустимые символы Windows, помечает дубли, пустые токены и обрезанные длинные имена;
- настройки профиля сохраняются в `%AppData%/TrueBIM/BimTools/batch-export/settings.json`;
- PDF экспорт выполняется через существующий `PrintPdfExportService`;
- DWG экспорт выполняется через существующий `PrintCadExportService`;
- окно ведет отчет по каждому листу/формату и умеет выгрузить отчет в CSV UTF-8 with BOM.

Manual QA:

1. Установить свежий `TrueBIM-Setup.exe` или выполнить local deploy после закрытия Revit.
2. Открыть проект Revit с несколькими печатаемыми листами.
3. Вкладка `TrueBIM` -> панель `Выпуск` -> `Экспорт PDF/DWG`.
4. Проверить список листов, фильтр по номеру/имени и preview имен.
5. Указать папку выгрузки, оставить `PDF`, выбрать 1-2 листа и нажать `Экспортировать`.
6. Проверить созданные `.pdf` файлы и строки на вкладке `Отчёт`.
7. Включить `DWG`, повторить экспорт на маленьком наборе листов и проверить `.dwg`.
8. Нажать `Отчёт CSV`, сохранить файл и проверить открытие CSV в Excel/текстовом редакторе.
9. Закрыть окно, снова открыть инструмент и проверить, что папка/шаблон/форматы восстановились из JSON.

## Задача 0.2: Диспетчер Семейств

Статус: готово.

Цель: дать безопасный браузер библиотек `.rfa`, который помогает найти семейство, сохранить избранное/историю и загрузить его в текущий проект без редактирования family-документов.

Фактически реализовано:

- команда `FamilyManagerCommand` запускается из панели `Библиотека`;
- окно `FamilyManagerWindow` заменяет заглушку `Диспетчер семейств`;
- можно добавить и удалить папки библиотек;
- сканер рекурсивно ищет `.rfa` в включенных папках и кэширует результат в `%AppData%/TrueBIM/BimTools/family-manager/settings.json`;
- есть поиск по имени, категории и пути;
- категории определяются эвристически по имени файла/папок, без открытия `.rfa`;
- есть фильтр категорий и режим показа только избранного;
- избранное и история загрузок сохраняются в JSON;
- если семейство уже загружено, инструмент спрашивает, перезаписать файлом библиотеки или использовать существующее семейство;
- загрузка семейства выполняется через Revit API внутри `Transaction`;
- кнопка `Загрузить и разместить` активирует первый или выбранный уже загруженный тип и запускает штатное размещение Revit через `PostRequestForElementTypePlacement`.

Manual QA:

1. Установить свежий `TrueBIM-Setup.exe` или выполнить local deploy после закрытия Revit.
2. Открыть проект Revit.
3. Вкладка `TrueBIM` -> панель `Библиотека` -> `Диспетчер семейств`.
4. Добавить папку с несколькими `.rfa` файлами и нажать `Сканировать`.
5. Проверить список семейств, поиск по имени, фильтр категории и переключатель `Избранное`.
6. Добавить одно семейство в избранное, закрыть и снова открыть окно; избранное должно восстановиться.
7. Выбрать семейство, нажать `Загрузить` и проверить появление семейства в Project Browser.
8. Повторить для уже загруженного семейства и проверить диалог подтверждения перезаписи.
9. Нажать `Загрузить и разместить`; Revit должен перейти в штатный режим размещения типа семейства.
10. Проверить, что история загрузок сохраняется после перезапуска окна.

## Задача 0.3: Оформить Штамп

Статус: готово.

Цель: безопасно заполнять выбранные параметры листов и размещённых на них title block instances по настраиваемым правилам с предпросмотром до записи.

Фактически реализовано:

- команда `TitleBlockFillCommand` запускается из панели `Оформление`;
- окно `TitleBlockFillWindow` заменяет заглушку `Оформить штамп`;
- список листов берётся через существующий `SheetCollectorService`;
- для каждого листа показывается статус найденного штампа;
- профиль правил хранится в `%AppData%/TrueBIM/BimTools/title-block-fill/settings.json`;
- правило задаёт цель `Лист` или `Штамп`, имя параметра, источник значения и дополнительное значение;
- поддержаны источники `Статический текст`, `Дата`, `Номер листа`, `Имя листа`, `Параметр проекта`, `Параметр листа`;
- `Предпросмотр` показывает текущее и новое значение, read-only параметры, отсутствующие параметры и неподдержанные storage types;
- `Применить` записывает значения внутри одного Revit `Transaction`;
- ошибки отдельных листов или параметров попадают в отчёт и не прерывают весь инструмент;
- отчёт можно сохранить в CSV UTF-8 with BOM.

Manual QA:

1. Установить свежий `TrueBIM-Setup.exe` или выполнить local deploy после закрытия Revit.
2. Открыть проект Revit с листами и размещёнными штампами.
3. Вкладка `TrueBIM` -> панель `Оформление` -> `Оформить штамп`.
4. Проверить список листов, фильтр и колонку `Штамп`.
5. На вкладке `Правила` добавить правило для безопасного текстового параметра листа, например `Комментарии`, со статическим текстом.
6. Добавить правило для параметра штампа, который точно существует в title block family.
7. Нажать `Предпросмотр` и проверить текущие/новые значения на вкладке `Отчёт`.
8. Нажать `Применить`, подтвердить запись и проверить отчёт по записанным/пропущенным/ошибочным строкам.
9. Проверить значения параметров листа и штампа в Revit, затем сделать Undo.
10. Закрыть и снова открыть окно; профиль правил должен восстановиться из JSON.

## Задача 0.4: Автомарки

Статус: готово.

Цель: безопасно поставить марки на элементы активного 2D-вида с предпросмотром, проверкой уже существующих марок и отчётом по каждому элементу.

Фактически реализовано:

- команда `AutoTagCommand` запускается из панели `Оформление`;
- окно `AutoTagWindow` заменяет заглушку `Автомарки`;
- инструмент работает только на обычном активном 2D-виде и отклоняет шаблоны, листы, browser views и 3D-виды;
- категории собираются из model-элементов, видимых на активном виде;
- можно выбрать категории, тип марки из загруженных tag family symbols или оставить режим `Авто (по категории)`;
- профиль сохраняется в `%AppData%/TrueBIM/BimTools/auto-tags/settings.json`;
- `Предпросмотр` показывает ElementId, категорию, имя, число существующих марок, статус и причину пропуска;
- по умолчанию включён режим `Только без марки`, который пропускает элементы с уже существующей маркой на активном виде;
- точка марки берётся из `LocationPoint`, середины `LocationCurve` или центра bounding box;
- `Поставить марки` создаёт `IndependentTag` внутри Revit `Transaction`;
- ошибки отдельных элементов попадают в отчёт и не прерывают весь инструмент;
- отчёт preview/apply можно сохранить в CSV UTF-8 with BOM.

Manual QA:

1. Установить свежий `TrueBIM-Setup.exe` или выполнить local deploy после закрытия Revit.
2. Открыть проект Revit с загруженными марками для стен, дверей или другого model-класса.
3. Открыть план, разрез или фасад с несколькими видимыми элементами.
4. Вкладка `TrueBIM` -> панель `Оформление` -> `Автомарки`.
5. Проверить список категорий и оставить одну безопасную категорию, например `Стены` или `Двери`.
6. Оставить тип `Авто (по категории)` или выбрать совместимый тип марки.
7. Нажать `Предпросмотр` и проверить строки `Готово` / `Пропущено`, особенно для элементов, у которых уже есть марки.
8. Выбрать 1-2 строки, нажать `Поставить марки`, подтвердить операцию и проверить появление марок на активном виде.
9. Проверить отчёт, сохранить CSV и открыть его в Excel/текстовом редакторе.
10. Сделать Undo в Revit, затем закрыть и снова открыть окно; профиль категорий и опции должны восстановиться из JSON.

Основное поведение:

- Использовать единственный предвыбранный элемент как исходный, если он есть; иначе попросить пользователя выбрать исходный элемент.
- Показывать копируемые параметры с именем, значением, storage type, источником instance/type, возможностью записи и предупреждениями.
- Давать выбрать элементы-получатели через Revit selection.
- Копировать значения внутри `Transaction`, учитывая `StorageType`, GUID shared parameter, built-in parameters, read-only флаги и совместимость типов.
- Показывать отчёт по скопированным значениям, пропускам и причинам пропуска.

Фактически реализовано:

- Команда `CopyParametersCommand` запускается из панели `Параметры`.
- Если выбран ровно один элемент, он используется как исходный; иначе команда просит выбрать исходный элемент.
- Окно `CopyParametersWindow` показывает доступные для копирования instance/type параметры, текущие значения, storage type и предупреждения для type/ElementId/опасных параметров.
- После выбора параметров Revit просит выбрать элементы-получатели.
- `ParameterCopyService` копирует значения внутри `Transaction` и ищет параметры-получатели по GUID shared parameter, `BuiltInParameter`, затем по имени.
- Поддержаны `String`, `Integer`, `Double`, `ElementId`; несовпадения типа, read-only параметры и отсутствующие параметры попадают в отчёт как пропуски.
- `Esc` при выборе элементов возвращает `Result.Cancelled`.

2. На панели `Параметры` нажать `Копирование параметров`.
3. Выбрать исходный элемент или запустить команду с одним заранее выбранным элементом.
4. Отметить несколько безопасных параметров, например текстовые комментарии или марку.

7. Проверить свойства элементов-получателей и Undo Revit.

## Задача 3: Рабочие Наборы

Планируемые файлы:

Основное поведение:

- Сначала поддержать CSV; `.xlsx` добавлять только если зависимость уже надёжно встроена в проект.
- Показывать предпросмотр нормализованных имён worksets до применения.
- Помечать строки как `Будет создан`, `Уже существует`, `Пустая строка`, `Недопустимое имя` или `Дубликат в файле`.
- Никогда не включать worksharing молча; предупреждать про undo history и требовать явное подтверждение.
- Создавать worksets внутри `Transaction` и показывать отчёт по созданным, пропущенным и ошибочным строкам.

Фактически реализовано:

- Команда `CreateWorksetsCommand` запускается из панели `Администрирование`.

- `WorksetValidationService` помечает строки как `Будет создан`, `Уже существует`, `Пустая строка`, `Недопустимое имя` или `Дубликат в файле`.

- `WorksetCreationService` создаёт worksets внутри `Transaction`, повторно проверяет уникальность имени через Revit API и показывает итоговый отчёт.

## Задача 4: Цвета По Параметрам

Статус: готово.

Цель: окрашивать элементы активного вида по значениям выбранного параметра через Revit view filters.

Планируемые файлы:

- `ColorByParameterCommand.cs`
- `ColorByParameterWindow`
- `ColorByParameterService`
- `ParameterValueReader`
- `ViewFilterService`
- `ColorPaletteService`
- `FilterNameBuilder`
- `BimCategoryItem`
- `BimParameterItem`
- `ColorRuleRow`
- `ColorApplyResult`

Основное поведение:

- Собирать категории из элементов, видимых на активном виде.
- Показывать параметры выбранных категорий с разделением по источнику и storage type.
- Показывать уникальные значения, включая `<Пусто / Не заполнено>`, с цветами и чекбоксом использования.
- Применять `ParameterFilterElement` и `OverrideGraphicSettings` только к активному виду.
- Использовать безопасные имена фильтров с префиксом `BIM_F_`.
- Ограничивать чрезмерное количество уникальных значений и предупреждать пользователя перед созданием большого числа фильтров.
- Очищать только фильтры активного вида, созданные этим инструментом, и не удалять пользовательские фильтры молча.

- Значения включают `<Пусто / Не заполнено>`; список ограничен первыми 40 уникальными значениями с предупреждением в статусе.
- `ViewFilterService` создаёт или обновляет `ParameterFilterElement` с безопасным именем `BIM_F_...` и применяет `OverrideGraphicSettings` только к активному виду.
- Для строковых, целых, числовых, `ElementId` и пустых значений создаются Revit filter rules; неподходящие сочетания попадают в отчёт как пропуски.
- Очистка снимает с активного вида только фильтры с префиксом `BIM_F_`, не удаляя сами элементы фильтров из проекта.
- Перед созданием более 25 фильтров показывается явное подтверждение.

## Задача 5: ParaManager MVP

Статус: готово.

Цель: импортировать shared parameters в проект из CSV или Excel с выбранным shared parameter file.

Планируемые файлы:

- `ParaManagerCommand.cs`
- `ParaManagerWindow`
- `SharedParameterFileService`
- `ProjectParameterBindingService`
- `ParameterExcelImportService`
- `CategoryResolveService`
- `ParameterExportService`
- `ParaManagerValidationService`
- `ParameterImportRow`
- `ParameterBindingPreviewRow`
- `ParameterImportResult`
- `SharedParameterDefinitionInfo`

Основное поведение:

- Начать с импорта project parameters из CSV/Excel.
- Требовать выбранный shared parameter `.txt`.
- До применения валидировать категории, типы данных, binding type, существующие definitions и конфликты привязки.
- Создавать или переиспользовать группы и definitions в shared parameter file.
- Привязывать instance/type параметры к найденным категориям внутри `Transaction`.
- Показывать отчёт по созданным definitions, привязанным параметрам, пропущенным строкам и ошибкам.

Фактически реализовано:

- Команда `ParaManagerCommand` запускается из панели `Параметры`.
- Окно `ParaManagerWindow` содержит вкладки `Параметры проекта`, `Импорт CSV` и `Отчёт`.
- Вкладка `Параметры проекта` читает реальные project parameter bindings из `Document.ParameterBindings` и показывает имя, тип, instance/type, категории, группу и GUID для shared parameters.
- Вкладка `Параметры проекта` экспортирует текущий список project parameters в CSV с колонками `ParameterName;BindingType;Categories;GroupUnder;DataType;IsShared;Guid`.
- Вкладка `Импорт CSV` поддерживает выбор CSV и shared parameter `.txt`, экспорт шаблона и предпросмотр строк.
- CSV MVP использует колонки `ParameterName;SharedGroup;BindingType;Categories;GroupUnder;DataType;Visible;UserModifiable;Description`.
- Поддержанные `DataType`: `Text`, `Integer`, `Number`, `YesNo`, `Length`, `Area`, `Volume`.
- Поддержанные `GroupUnder`: `Identity Data`, `Text`, `Dimensions`, `Data`, `General`, `Other`.
- `ParaManagerValidationService` до применения проверяет имя, дубли, binding type, категории, тип данных, группу свойств и boolean-флаги.
- `SharedParameterFileService` открывает выбранный `.txt`, создаёт группу и `ExternalDefinition`, если их нет.
- `ProjectParameterBindingService` создаёт instance/type binding, добавляет параметр через `BindingMap.Insert`, а для существующего параметра с тем же binding type расширяет категории через `ReInsert`.
- При конфликте instance/type строка не ломает модель, а попадает в отчёт как ошибка.
- Все изменения project parameters выполняются внутри `Transaction`.

## Общие Правила

- Все изменения модели выполнять внутри Revit `Transaction`.
- При отмене пользовательского выбора возвращать `Result.Cancelled`, где используется Revit selection.
- Ошибки показывать через `TaskDialog` и писать в лог TrueBIM.
- Для автоматически созданных фильтров, настроек или definitions использовать префикс `BIM_`.
- Не удалять пользовательские фильтры, параметры или рабочие наборы без явного подтверждения.
