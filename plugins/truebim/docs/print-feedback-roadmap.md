# Печать: RoadMap исправлений по пользовательским замечаниям

Статус документа: 2026-07-17.

## Источники и подтвержденные причины

- Замечания: `Печать листов замечание.docx`, 17 отдельных пунктов и 5 скриншотов.
- Диагностический лог: `truebim.txt`, 300 строк пользовательского сценария Revit 2023.
- На 95 листах первичная подготовка заняла `268955 ms`, повторные полные чтения — `242149 ms`, `255808 ms` и `240697 ms`.
- Обновление уже загруженных строк занимало `32-37 ms`. Значит основная задержка была в графикозависимом обходе Revit-листов, а не в WPF-таблице или маске имени.
- DWF стабильно падал с `Exporting to DWF/DWFX requires that document is modifiable.`
- Объединенный DWG в одном из повторных запусков создался, но получил служебное имя `<источник>_DWG.dwg`, не собранное по пользовательской маске. Поэтому замечания про `Один DWG` и лишнюю информацию в имени рассматриваются вместе как дефект контракта результата, а не только как единичный сбой API.

## Этапы реализации

### Этап 1. Быстрое открытие и ленивые проверки — выполнен в коде

- Первичный сбор не обращается к `ViewSheet.Outline` и `ViewSheet.CanBePrinted`.
- Непечатаемые листы определяются только среди выбранных перед фактическим запуском.
- Смена пресета использует уже загруженные параметры и не запускает принудительное полное чтение.
- В лог добавлено время проверки выбранных листов и число отклоненных листов.

Критерии приемки:

- открытие списка из 95 листов больше не зависит от формирования графики всех листов;
- пресеты с уже известными токенами переключаются без повторного чтения Revit-документа;
- если новый пресет требует новый параметр листа, перечитывается только нужный источник;
- перед экспортом `CanBePrinted` проверяется только у выбранных листов;
- ручной прогон в пользовательском RVT подтверждает отсутствие пауз по 4-5 минут.

### Этап 2. Исправление результата экспорта

- DWF в корректном модифицируемом контексте — выполнено в коде: используется существующая транзакция либо временная транзакция с обязательным откатом;
- детерминированный `Один DWG` — выполнено в коде: один файл на документ-источник, отдельная сохраняемая маска, preview итогового имени, без служебного `_DWG`;
- одинаковые имена общих DWG для разных документов блокируются до запуска с предложением добавить `{Имя документа}` или параметр проекта;
- единая маска для отдельных и общих PDF/DWG — выполнено в коде: общие PDF и DWG используют один token-template механизм и отдельные сохраняемые маски;
- общий PDF больше не получает автоматический префикс документа; для нескольких источников одинаковые итоговые имена блокируются до запуска;
- проверка существующих файлов по фактическим итоговым именам — выполнено в коде: preview, preflight и Revit export получают одну рассчитанную карту имен по источникам;
- итоговый отчет использует фактические `ExportedFiles` всех PDF/CAD-сервисов: кратко показывает первые три полных пути, а весь список созданных файлов и ошибок доступен в подробностях;
- тесты имен и матрица ручного QA PDF/DWG/DWF.

### Этап 3. Конструктор масок и параметры основной надписи — выполнен в коде

- рядом с каждой маской добавлен picker `Токены` с системными токенами и каталогами параметров листа, основной надписи и сведений о проекте;
- выбранный токен вставляется в текущую позицию курсора; для параметрического токена автоматически ставится в очередь ленивое перечитывание только нужных значений;
- добавлены русские и английские контракты `Параметр основной надписи:<имя>` / `TitleBlockParameter:<name>`;
- формат листа определяется по `SHEET_WIDTH`/`SHEET_HEIGHT` основной надписи без обращения к `ViewSheet.Outline`;
- preview отдельных и общих имён использует значения основной надписи наравне с параметрами листа и проекта;
- каталоги кэшируются по источнику, определения основной надписи — по её типу, а значения читаются только для токенов текущих масок.

### Этап 4. Таблица и выбор листов

- natural sort номеров — выполнен в коде для первичного сбора и WPF-таблицы; составные номера сравниваются по каждому числовому фрагменту, дубли стабилизируются именем листа и ElementId;
- многоуровневая группировка — выполнена в коде: при нескольких видимых источниках таблица строит уровни `источник -> том/группа -> лист`, при фильтре одного источника скрывает лишний верхний уровень;
- `Shift+Click` для диапазона чекбоксов — выполнен в коде: состояние опорного чекбокса применяется ко всему прямому или обратному диапазону в текущем порядке таблицы, непечатаемые строки остаются выключенными;
- гарантированный вертикальный скроллбар — выполнен в коде: секция листов использует ограниченную `Grid`-разметку вместо бесконечного измерения через `StackPanel`, полоса прокрутки видна постоянно, виртуализация строк и групп сохранена;
- понятное название листов-заглушек — выполнено в коде: переключатель называется `Неразмещенные листы (заглушки)`, tooltip объясняет связь с предварительными строками ведомости, статус прямо сообщает, что строка не печатается, а значение по умолчанию остается выключенным.

### Этап 5. Настоящая печать — выполнен в коде

- добавлен явный переключатель `Экспорт в файлы / Печать на принтер`; стартовым остается экспорт, а не относящиеся к выбранному режиму поля скрываются;
- список принтеров читается из установленных устройств Windows, текущий принтер Revit выбирается при наличии;
- диапазон зафиксирован как отмеченные в таблице листы и исполняется по одному листу через `PrintManager.SubmitPrint(View)` без временного `ViewSheetSet` в модели;
- доступны текущая настройка каждого документа и именованные Revit print setups; при нескольких источниках именованная настройка проверяется отдельно в каждом документе;
- добавлен отдельный журнал конфигурации драйвера, отправки каждого листа, ошибок и восстановления исходных глобальных настроек печати Revit;
- PDF-драйвер помечается в списке, показывает постоянное предупреждение и требует подтверждения перед запуском;
- сохранение принтера в пресете только после подтверждения UX и совместимости версий Revit.

### Этап 6. Финальный UX и релизная проверка

- единый шаблон переключателей PDF/DWG — выполнено в коде: `PDF + Один PDF` и `DWG + Один DWG`; трехрежимный PDF ComboBox удален, старые настройки `отдельные + общий` безопасно сворачиваются в `Один PDF`;
- опция открытия папки результата — выполнено в коде: настройка сохраняется в последнем состоянии и пресетах, Проводник открывается один раз после итогового сообщения и только при наличии хотя бы одного фактически созданного файла;
- краткие подсказки и итоговый отчет с путями — выполнено в коде: элементы управления объяснены через tooltip, итоговый статус различает успешный, частично успешный и неуспешный экспорт, показывает до трех фактических путей и раскрывает полный список при большом пакете;
- ручной прогон на 95+ листах в Revit 2022, 2023 и 2025;
- обновление installer только после прохождения сценариев этапов 1-5.

## Следующая задача

Выполнить ручной прогон модуля на 95+ листах в Revit 2022, 2023 и 2025: проверить экспорт PDF/DWG/DXF/DWF, физическую печать, симметричные переключатели, итоговый отчет и открытие папки результата.
