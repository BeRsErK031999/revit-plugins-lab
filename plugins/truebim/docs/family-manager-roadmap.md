# Roadmap: Диспетчер семейств

Дата среза: 2026-07-08.

## Что считается аналогом Family Manager

Ориентир для TrueBIM: браузер библиотек `.rfa` внутри Revit, который показывает пользовательские папки семейств, категории, семейства, типы, параметры выбранного типа, поиск, избранное, быстрый load/place и служебные действия вокруг файла.

Референсы:

- Autodesk App Store, `Family Manager` от ZXSoft: dock panel, папки пользователя, browsing/search, double-click/context menu для размещения экземпляра.
  <https://apps.autodesk.com/RVT/en/Detail/HelpDoc?appId=1541666128657400640&appLang=en&os=Win64>
- RVT Plugins review: категории, families, types, parameters, locate in Windows Explorer, search highlight, thumbnails, favourites, background loading.
  <https://rvtplugins.com/posts/zxsoft/family-manager/1541666128657400640>
- DiRoots FamilyReviser как смежный ориентир администрирования контента: import/export в локальные репозитории, очистка backup-файлов, prefix/suffix, find/replace, worksets.
  <https://diroots.com/revit-plugins/manage-revit-families-and-worksets-with-familyreviser/>
- Autodesk Revit Load Families: штатная загрузка `.rfa` и preview family content остаются базовым поведением Revit, поверх которого строится TrueBIM UI.
  <https://help.autodesk.com/view/RVT/2024/ENU/?guid=GUID-2A6C26DE-63D7-48E8-A986-CAB6C362049A>

## Текущий статус TrueBIM

Уже реализовано:

- команда `Диспетчер семейств` на панели `Библиотека`;
- список пользовательских папок библиотек;
- рекурсивный scan `.rfa`;
- JSON-cache в `%AppData%/TrueBIM/BimTools/family-manager/settings.json`;
- поиск по имени, категории, типу и пути;
- фильтр категорий;
- избранное и история загрузок;
- чтение реальной категории и типов из `.rfa` через `Application.OpenDocumentFile`;
- загрузка семейства через `Document.LoadFamily`;
- подтверждение overwrite для уже загруженного семейства;
- `Загрузить и разместить` через `PostRequestForElementTypePlacement`;
- чтение и отображение параметров выбранного типа из family-документа.
- дерево структуры `Библиотека -> Категория -> Семейство -> Тип`;
- контекстные действия `Открыть в Explorer` и `Скопировать путь`.
- пакетное обновление метаданных по выбранной папке библиотеки.
- thumbnail-cache в `%AppData%/TrueBIM/BimTools/family-manager/thumbnails`;
- preview выбранного семейства в правой панели с безопасным placeholder-состоянием;
- ручное обновление preview выбранного `.rfa` через `ElementType.GetPreviewImage`.
- явные источники библиотеки: отдельные кнопки `Папка` и `Файл`, без выбора папки через file picker;
- preset-колонки `Ширина`, `Высота`, `Материал` в списке семейств;
- фильтр параметров выбранного типа по тексту и preset-группам.

## Что нужно доделать

### 0. Пакетное обновление метаданных

Статус: готово.

- Обновлять категорию, типы и параметры всех семейств из выбранной папки.
- Перед запуском показывать подтверждение с количеством `.rfa`.
- Сохранять обновлённые метаданные в JSON-cache одним проходом.
- Показывать итог: обновлено, ошибки/пропуски.

### 1. Метаданные и параметры

Статус: в работе, основной browsing-срез готов.

- Сохранять параметры каждого типа в metadata-cache.
- Показывать параметры выбранного типа в правой панели.
- Искать семейства по имени, значению и формуле параметра.
- Доработать форматирование `Double` через display units Revit, чтобы вместо внутренних единиц показывать пользовательские единицы проекта.
- Добавить фильтр по имени параметра и preset-колонки для частых параметров.
  Статус: готово для `Ширина`, `Высота`, `Материал` и preset-групп правой панели.

### 2. Навигация как у content browser

Статус: первый срез готов.

- Добавить дерево `Папка -> Категория -> Семейство -> Тип`.
- Сохранить режимы отображения: дерево, плоский список, избранное, недавно загруженные.
- Добавить контекстные действия: открыть папку в Explorer, скопировать путь, обновить метаданные для папки.
- Подсвечивать найденные совпадения в имени семейства, типе и параметрах.
- Добавить понятный выбор источника библиотеки: кнопка `Папка` открывает folder picker, кнопка `Файл` добавляет отдельный `.rfa`.
  Статус: готово.

### 3. Thumbnails и preview

Статус: первый срез готов.

- Извлекать или генерировать thumbnail для `.rfa`.
- Кэшировать thumbnail рядом с metadata-cache.
- Показывать preview выбранного семейства в правой панели.
- Обрабатывать семейства без preview безопасным placeholder-состоянием.

### 4. Загрузка конкретного типа

Статус: частично закрыто через выбор уже загруженного/кэшированного имени типа.

- Проверить поведение type catalog `.txt` рядом с `.rfa`.
- Для семейств с каталогом типов дать пользователю выбрать тип до загрузки.
- После загрузки подтверждать, что выбранный тип реально появился в проекте.
- Если точечная загрузка типа невозможна через API для конкретного семейства, явно показывать fallback на полную загрузку семейства.

### 5. Администрирование библиотеки

Статус: не начато.

- Поиск дублей по имени, размеру, дате и относительному пути.
- Проверка отсутствующих файлов и устаревшего cache.
- Очистка `.00xx.rfa` backup-файлов как отдельная безопасная команда с preview.
- Экспорт списка библиотеки в CSV.
- Базовые правила качества: пустая категория, отсутствующие типы, неизвестная версия/ошибка открытия.

### 6. Пакетные операции над семействами

Статус: отложено до ручной проверки read-only сценариев.

- Переименование family/type с preview и откатом плана.
- Prefix/suffix и find/replace для имён семейства и типов.
- Экспорт загруженных семейств из проекта в выбранную библиотеку.
- Workset-правила для загруженных семейств, если это подтвердится как нужный сценарий.

## Ближайший порядок работ

1. Протестировать type catalog и поведение `LoadFamily` на реальных семействах.
2. Доработать форматирование `Double` через display units Revit.
3. Вернуться к пакетному администрированию только после подтверждения UX в Revit.

## Manual QA

1. Закрыть Revit перед local deploy или installer smoke.
2. Открыть проект Revit.
3. `TrueBIM` -> `Библиотека` -> `Диспетчер семейств`.
4. Нажать `Папка`, выбрать папку библиотеки через folder picker, убедиться что кнопка выбора подтверждает текущую папку.
5. Нажать `Файл`, выбрать отдельный `.rfa`, нажать `Сканировать`.
6. Выбрать семейство и нажать `Обновить метаданные`.
7. Проверить список `Типы в файле/проекте`.
8. Выбрать тип и проверить таблицу `Параметры выбранного типа`.
9. Отфильтровать параметры текстом и preset-группами `Размеры`, `Материалы`, `Идентификация`.
10. Проверить preset-колонки `Ширина`, `Высота`, `Материал` в списке семейств.
11. Найти семейство поиском по имени параметра или значению параметра.
12. Проверить дерево структуры: библиотека, категория, семейство и тип должны выбирать ту же строку в таблице.
13. В контекстном меню дерева или таблицы нажать `Открыть в Explorer` и `Скопировать путь`.
14. Выбрать папку библиотеки и нажать `Метаданные папки`; подтвердить пакетное обновление и проверить итоговый статус.
15. Закрыть и открыть окно снова: типы и параметры должны восстановиться из JSON-cache.
16. Нажать `Обновить preview` и проверить, что preview появился в правой панели либо показан безопасный placeholder.
17. Закрыть и открыть окно снова: preview должен восстановиться из thumbnail-cache.
18. Нажать `Загрузить` и `Загрузить и разместить` на тестовом семействе.
