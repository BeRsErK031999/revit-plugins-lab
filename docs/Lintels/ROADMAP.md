# Перемычки — RoadMap

## Цель

Создать во вкладке `КР` понятный пошаговый инструмент `Перемычки`, который позволяет пакетно обработать несколько явно отмеченных типоразмеров и довести каждый до готового изображения для спецификации:

1. пользователь выбирает, откуда взять перемычки;
2. отмечает один или несколько типоразмеров;
3. одним подтверждением создаёт недостающие сборки для отмеченных строк;
4. выбирает загружаемое семейство рамки `.rfa` и создаёт или обновляет боковые виды `1:10`;
5. получает оформленный вид, локальный PNG и понятный отчёт с именами и путями;
6. при необходимости обновляет изображение в выбранном параметре типа или экземпляра.

Контракт уточнён по встрече и ответам пользователя 15–16.07.2026. Решения из раздела ниже считаются целевыми, пока пользователь явно их не изменит.

## Подтверждённый пользовательский сценарий

- Основной интерфейс — крупный растягиваемый модальный пошаговый мастер с русскими, нетехническими формулировками.
- Первый шаг предлагает режимы `Текущее выделение`, `Активный вид`, `Весь проект` и `Результаты, созданные TrueBIM`.
- Кандидат определяется по семейству. Вложенные компоненты и уголки входят в состав перемычки.
- Пользователь выбирает один или несколько типоразмеров. Для каждого типоразмера по-прежнему используется один детерминированный экземпляр-представитель; автоматическая обработка всех экземпляров одного типа не запускается.
- Несколько отмеченных строк проходят одну пакетную проверку и одно подтверждение. Ошибка одной строки не отменяет уже обработанные строки.
- Для каждого отмеченного типоразмера создаются отдельные сборка, боковой вид и PNG.
- Если сборка, вид или изображение уже существуют, плагин спрашивает, что сделать: открыть, пропустить, обновить или пересоздать.
- Ранее созданные неправильные элементы не исправляются без отдельного согласия пользователя.
- Пользователь сам выбирает, какие этапы выполнить: сборка, вид, оформление, марки, PNG и назначение изображения параметру.
- Отдельный технический preflight пользователю не показывается: безопасная проверка выполняется внутри основной кнопки.
- После операции мастер показывает имена, `ElementId`, путь к PNG, предупреждения и итог по каждому выбранному этапу.
- Независимая ошибка одного этапа не должна скрывать результаты успешно завершённых этапов.

## Подтверждённый результат оформления

- Один выбранный экземпляр перемычки со своим составом показывается на одном боковом виде.
- Масштаб вида — `1:10`.
- Ориентация определяется автоматически; пользователь может выбрать `Слева` или `Справа`.
- Шаблон вида выбирает пользователь, включая вариант `Без шаблона`.
- Базовые настройки без шаблона: детализация `Высокая`, стиль `Скрытые линии`, дисциплина `Координация`.
- Геометрия центрируется внутри области.
- Минимальная область — `1050 × 385 мм` в координатах модели; для длинной перемычки область увеличивается без обрезки.
- В итоговом PNG видна рамка из явно выбранного пользователем загружаемого семейства категории `Типовая аннотация`. Плагин не подбирает семейство по части имени и не строит рамку detail lines.
- Создаётся один габаритный размер общей длины над геометрией.
- Высотная отметка ставится по верхней грани сбоку от геометрии.
- Аннотации по возможности автоматически раздвигаются при пересечении.
- Марки располагаются снизу и показывают тип вложенного компонента. При отсутствии подходящей марки или значения плагин показывает предупреждение вместо `?`.
- Материал стены, опирание и составной эскиз из двух видов пока не создаются.

## Изображение и параметр Revit

- Формат — PNG с белым фоном.
- Начальный фиксированный canvas — `1600 × 587 px`; для расширенной области сохраняется тот же размер изображения.
- Локальный каталог: `%LOCALAPPDATA%\TrueBIM\Lintels\<ProjectName>`.
- Имя: `TB_Перемычка_<Семейство>_<Тип>_<ElementId>.png` после очистки недопустимых символов.
- Повторный запуск обновляет существующий PNG.
- Импорт изображения в Revit включается отдельной опцией.
- Пользователь выбирает совместимый изменяемый параметр типа или экземпляра.
- Проверка конкретной спецификации в текущий объём не входит.

## UX и терминология

Целевые шаги мастера:

1. `Откуда взять перемычки`;
2. `Выберите типоразмеры`;
3. `Создайте сборки`;
4. `Выберите рамку и создайте виды`.

Основные разделы: `Типоразмеры`, `Что создаст TrueBIM`, `Семейство рамки`, `Результат`.

Таблица выбора должна показывать: статус, семейство, тип, уровень, `ElementId`, существующую сборку, вид и изображение.

Термины в интерфейсе:

- `Preflight` → `Проверить перед созданием` либо скрытая автоматическая проверка;
- `Артефакты` → `Что создаст TrueBIM`;
- `Crop` → `Границы изображения`;
- `FamilyInstance` → `Экземпляр перемычки`;
- `Diagnostics` → `Почему элемент пропущен`.

Статусы имеют цвет и текст: `Готово`, `Нужно внимание`, `Ошибка`, `Уже существует`. Все отключённые действия получают русскую tooltip-подсказку.

## Технические границы

- Код остаётся внутри `TrueBIM.App.Modules.Lintels`, кроме тонкой команды запуска.
- Чтение документа и подготовка выбора не изменяют модель.
- Операции одного экземпляра объединяются в управляемую группу транзакций, чтобы пользователь мог отменить их одной командой Undo.
- TrueBIM хранит связь только со своими сборками, видами, экземплярами семейства рамки и изображениями.
- В журнал записываются созданные, обновлённые, открытые, пропущенные и ошибочные элементы с `ElementId`.
- Целевая совместимость — Revit 2019–2026. Приёмка выполняется по доступным локальным версиям и отдельно фиксирует отсутствующие API.
- Обычный проект содержит около 50 перемычек; мастер обрабатывает пакет явно отмеченных типоразмеров, по одному детерминированному экземпляру-представителю на тип.

## Уже реализованная основа

- Кнопка `Перемычки` во вкладке `КР`.
- Поиск выделенных экземпляров и кандидатов на активном виде.
- Группировка по Revit `TypeId` и диагностика вложенной геометрии.
- Детерминированные имена сборки, бокового вида и PNG.
- Проверка допустимости состава через Revit API.
- Создание одной Assembly и защита от дубликатов.
- Создание одного assembly view `ElevationLeft` в масштабе `1:10`.
- Один габаритный размер, Spot Elevation и нормализация области.
- Связь собственных элементов оформления через Extensible Storage.
- Базовые тесты чистой логики и реальная проверка на предоставленном RVT в Revit 2023.

Существующее окно и операции считаются рабочей технической основой, но не окончательным UX.

## Этапы v2

### 1. Фундамент модального мастера — выполнено в коде, требуется визуальная проверка в Revit

- Добавить модальный первый шаг `Откуда взять перемычки`.
- Показать четыре режима с понятным описанием и состоянием доступности.
- Рабочими в первом срезе оставить `Текущее выделение` и `Активный вид`.
- Если выделение пустое, отключить этот режим и объяснить причину.
- `Весь проект` показать как следующий этап; `Результаты, созданные TrueBIM` активировать только при наличии в проекте хотя бы одной Assembly с TrueBIM-именем.
- Передать выбранный источник в существующий read-only сбор данных.
- После подтверждения временно открывать существующее рабочее окно как следующий шаг, не ломая создание сборки и вида.
- Покрыть каталог режимов и выбор режима unit-тестами.

Критерий готовности: перед рабочим окном открывается модальный шаг с четырьмя режимами; пользователь понимает, какой источник выбран и почему отдельный режим недоступен.

Реализовано:

- команда открывает принадлежащий Revit модальный шаг `Откуда взять перемычки`;
- при наличии выделения по умолчанию выбран режим `Текущее выделение`, иначе — `Активный вид`;
- пустое выделение получает понятное объяснение и не может быть выбрано;
- `Весь проект` виден заранее и честно отмечен как недоступный;
- `Результаты, созданные TrueBIM` объясняют, что речь идёт только о сборках этого плагина, показывают `Пока пусто` до первого создания и становятся доступны после появления TrueBIM Assembly;
- выбранный режим передаётся в read-only сбор данных и сохраняется при обновлении списка;
- текущие рабочие операции создания сборки и вида остаются доступными после перехода;
- чистая логика выбора покрыта unit-тестами.

### 2. Пакет типоразмеров и один представитель каждого типа

- Подключить поиск по всему проекту.
- Показывать типоразмеры и детерминированный экземпляр-представитель каждого типа.
- Добавить уровень, `ElementId` и состав экземпляра.
- Разрешить выбор одного или нескольких типоразмеров.
- Выполнять единый preflight пакета и отдельную операцию создания для каждой строки.
- Сохранять все отметки после обновления данных из Revit.

Критерий готовности: пользователь одним запуском создаёт сборки для нескольких отмеченных типоразмеров и видит отдельный результат по каждой строке.

### 3. Понятная навигация по шагам 2–4

- Объединить выбор, настройки, выполнение и результат в одном модальном мастере.
- Добавить навигацию `Назад` / `Далее` без потери выбора.
- Постоянно показывать, что шаг 2 — выбор строк, шаг 3 — сборки, шаг 4 — семейство рамки и виды.
- Скрыть технические проверки за основной кнопкой.
- Перенести диагностику в разговорные предупреждения рядом с проблемной строкой.

### 4. Новые и результаты, созданные TrueBIM

- Добавить источник `Результаты, созданные TrueBIM`.
- Находить связанные TrueBIM Assembly, виды и изображения.
- Предлагать `Открыть`, `Пропустить`, `Обновить оформление`, `Пересоздать`.
- Никогда не пересоздавать существующие элементы без подтверждения.

### 5. Создание вида и настройки оформления

- Автоматически выбирать более информативную ориентацию `Слева/Справа`.
- Добавить ручной выбор ориентации.
- Добавить выбор шаблона вида или `Без шаблона`.
- Без шаблона применять `Высокая`, `Скрытые линии`, `Координация`.
- Показывать созданное имя и `ElementId` вида.

### 6. Загружаемое семейство рамки и базовые аннотации

- Дать пользователю выбрать один файл семейства рамки `.rfa` для всего пакета.
- Автоматически подставлять `•• (Аннотация) Рамка аннотации.rfa` из общей библиотеки `Y:\01 - REVIT\03 - FAMILIES 2017\Аннотации`, если путь доступен; при недоступной библиотеке не угадывать замену, а попросить выбрать файл.
- Загружать выбранное семейство в проект и размещать один экземпляр по центру каждого вида.
- Не использовать нечёткий поиск семейства по имени и не строить рамку detail lines.
- Проверять, что выбранное семейство можно разместить как аннотацию на виде, и показывать понятную ошибку для неподходящего файла.
- Перенести общий размер над геометрией.
- Стабилизировать Spot Elevation сбоку от верхней грани.
- Добавить расчёт свободных зон и раздвижение аннотаций.
- Расширять область для длинных перемычек и сохранять центрирование.

### 7. Марки компонентов

- Предлагать совместимые типы марок для видимых вложенных компонентов и уголков.
- По умолчанию показывать тип компонента.
- Размещать марки снизу и исключать наложения.
- При отсутствии ресурса или значения показывать предупреждение без `?`.

### 8. Экспорт PNG

- Экспортировать вид по границам рамки в белый PNG `1600 × 587 px`.
- Создавать локальный каталог проекта и безопасное имя с `ElementId`.
- Обновлять файл при повторном запуске.
- Проверять, что изображение не пустое и имеет ожидаемый canvas.
- Показывать путь и кнопку открытия каталога на финальном шаге.

### 9. Импорт и назначение параметру

- Найти совместимые параметры изображения у типа и экземпляра.
- Дать пользователю выбрать область параметра и конкретный параметр.
- Импортировать или обновить ImageType без дубликатов.
- Не изменять модель, если параметр неоднозначен или недоступен для записи.

### 10. Надёжность и восстановление

- Объединять операции одного экземпляра в одну отменяемую группу.
- Сохранять частичный понятный результат при независимой ошибке экспорта или назначения параметра.
- Добавить подробный итог и лог с `ElementId`, именами и путями.
- Проверить повторный запуск после частично завершённой операции.

### 11. Совместимость, визуальная QA и выпуск

- Проверить сборки Revit 2019–2026 по доступным API.
- Пройти ручной сценарий на предоставленном RVT.
- Сравнить итоговый PNG с референсом: одна перемычка, одна рамка, размер сверху, отметка сбоку, марки снизу.
- Проверить локальное развёртывание и installer flow из чистого коммита.
