# Перемычки — RoadMap

## Цель

Создать во вкладке `КР` инструмент `Перемычки`, который по выбранным экземплярам или типоразмерам перемычек формирует повторяемое оформление для спецификации:

1. находит перемычки и их вложенные компоненты;
2. создаёт одну сборку на выбранный типоразмер;
3. создаёт боковой вид сборки в масштабе `1:10`;
4. размещает размеры, высотную отметку, рамку и марки компонентов;
5. экспортирует одинаково кадрированные изображения;
6. импортирует изображения в проект и назначает их согласованному параметру перемычки.

RoadMap составлен по встрече от 15.07.2026. Спорные значения не считаются утверждённым контрактом до проверки на рабочем RVT-файле.

## Границы первого MVP

- Обрабатывается один открытый документ и один выбранный пользователем набор перемычек.
- Основной сценарий начинается с активного плана этажа, но источник элементов должен быть явно виден пользователю.
- Один представитель типоразмера используется для создания сборки и изображения.
- Геометрия может находиться во вложенных семействах: родительская перемычка не обязана содержать собственную геометрию.
- Первый MVP не строит материал стен вокруг перемычки и не моделирует опирание, пока требования не подтверждены.
- Повторный запуск не должен бесконтрольно создавать дубликаты сборок, видов и изображений.

## Не подтверждённые входные данные

До реализации операций записи нужно получить и проверить:

- рабочий RVT-файл с перемычками и вложенными компонентами;
- правило определения перемычки: категория, семейство, общий параметр или другое устойчивое свойство;
- семейство аннотации рамки;
- семейство марки компонентов;
- семейство или тип высотной отметки;
- тип линейного размера;
- точный размер рамки на листе. На встрече звучали `1050 × 385` в модели и ориентир около `105 × 38 мм` на бумаге при масштабе `1:10`; значение нужно измерить в образце;
- целевой параметр изображения: `Изображение типоразмера` у типа или `Изображение` у экземпляра;
- формат, разрешение, фон и каталог временного экспорта;
- правило именования сборок, видов и импортированных изображений;
- поддерживаемые версии Revit для приёмки.

## Архитектурные границы

- Код модуля располагается в `TrueBIM.App.Modules.Lintels`.
- Команда ribbon остаётся тонкой точкой входа; сбор данных, создание сборок, оформление, экспорт и назначение параметров разделяются на сервисы.
- Анализ документа и preview выполняются без транзакции.
- Каждая операция записи выполняется в отдельной управляемой транзакции с понятным результатом и логированием.
- Сначала поддерживается один подтверждённый тип семейства из образца, затем добавляются расширяемые правила распознавания.

## Этапы

### 1. Изолированный каркас модуля — выполнено

- Добавить кнопку `Перемычки` во вкладку `КР`.
- Добавить команду `LintelsCommand` без изменения модели.
- Добавить namespace `TrueBIM.App.Modules.Lintels` и статус первого этапа.
- Зафиксировать границы MVP, вопросы и последовательность работ в этом RoadMap.
- Покрыть регистрацию кнопки и безопасный статус модульными тестами.

Критерий готовности: TrueBIM собирается, кнопка зарегистрирована в `КР`, команда сообщает статус каркаса и не открывает транзакцию Revit.

### 2. Диагностический сбор перемычек

- Поддержать выбранные элементы и сбор кандидатов с активного плана.
- Зафиксировать правило определения родительской перемычки.
- Получать вложенные компоненты, реально содержащие геометрию.
- Группировать кандидатов по устойчивому ключу семейства/типоразмера.
- Показать диагностический отчёт без изменения модели.

Критерий готовности: на тестовом RVT пользователь видит найденные типоразмеры, количество экземпляров и причины исключения элементов.

### 3. Окно выбора и preview

- Показать типоразмеры с количеством экземпляров и источником выбора.
- Разрешить выбрать типоразмеры для обработки.
- Показать будущие имена сборок, видов и изображений.
- Заблокировать запуск при отсутствии вложенной геометрии или обязательных ресурсов.

Критерий готовности: до транзакции пользователь понимает, какие объекты и артефакты будут созданы.

### 4. Создание сборок

- Выбирать один репрезентативный экземпляр каждого типоразмера.
- Создавать сборку из вложенных компонентов, а не из пустого родительского контейнера.
- Проверять допустимость набора через Revit API до записи.
- Реализовать устойчивое именование и стратегию повторного запуска: пропуск, обновление или пересоздание только артефактов TrueBIM.

Критерий готовности: для поддерживаемого типоразмера создаётся ровно одна корректная сборка без дубликатов.

### 5. Боковой вид сборки

- Создавать боковой assembly view в правильной ориентации.
- Устанавливать масштаб `1:10`.
- Применять согласованный шаблон вида, detail level и visual style.
- Настраивать crop region по геометрии и рамке.

Критерий готовности: вид повторяет подтверждённый вручную пример и стабильно показывает вложенные компоненты.

### 6. Базовое оформление

- Размещать согласованный линейный размер.
- Размещать высотную отметку.
- Вставлять рамку и центрировать эскиз внутри неё.
- Не добавлять материал стен и опирание до отдельного подтверждения.

Критерий готовности: все изображения имеют одинаковую область кадрирования и сопоставимый визуальный масштаб.

### 7. Марки компонентов

- Подключить предоставленное семейство марки.
- Определить, какие вложенные обобщённые модели маркируются.
- Исключить дубли и разместить марки без наложений для поддерживаемого образца.

Критерий готовности: обозначения компонентов соответствуют эталонному эскизу.

### 8. Экспорт изображения

- Экспортировать изображение по границам рамки с фиксированными настройками.
- Использовать временный каталог TrueBIM и безопасные имена файлов.
- Проверять, что итоговое изображение не пустое и имеет ожидаемые размеры.
- Очищать временные файлы после успешного импорта.

Критерий готовности: изображения разных типоразмеров имеют одинаковый canvas и пригодны для строк спецификации.

### 9. Импорт и назначение параметру

- Импортировать или переиспользовать ImageType без дубликатов.
- Назначать изображение подтверждённому параметру типа или экземпляра.
- Не изменять семейство и тип при отсутствии однозначного параметра.
- Выводить отчёт по назначенным и пропущенным элементам.

Критерий готовности: изображение отображается в тестовой спецификации и сохраняется после повторного открытия проекта.

### 10. Повторный запуск и восстановление после ошибок

- Идентифицировать артефакты, созданные TrueBIM.
- Обновлять только принадлежащие модулю сборки, виды и изображения.
- Откатывать незавершённый типоразмер без потери результатов уже завершённых типов.
- Формировать итоговый отчёт и подробный лог.

### 11. Совместимость, QA и выпуск

- Добавить unit-тесты чистой логики группировки, именования и планирования.
- Добавить manual QA для поддерживаемых версий Revit.
- Проверить сборку, локальное развёртывание и установленный payload.
- Включить модуль в существующий installer flow без отдельных внешних зависимостей.

## Следующая задача

Этап 2: на предоставленном RVT-файле реализовать read-only диагностику родительских перемычек и вложенных компонентов. До получения файла и устойчивого правила идентификации операция создания сборок намеренно не начинается.
