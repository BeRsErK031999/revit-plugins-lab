# Армирование по изополям: RoadMap до рабочего плагина

Обновлено: 16.07.2026.

## Цель

Превратить текущий экспериментальный модуль в управляемый инструмент для армирования ЖБ-плит по расчётным картам требуемой площади арматуры. Пользователь должен загрузить комплект изополей, проверить распознанные зоны, связать их с плитой Revit, настроить правила перевода `см²/м` в реальные стержни и получить проверяемую раскладку с возможностью безопасного повторного запуска.

Первый производственный сценарий ограничивается простыми горизонтальными плитами. Стены, наклонные плиты, криволинейные элементы и сложные пространственные поверхности идут после стабилизации плит.

## Исходные материалы

Для проектирования сценария просмотрены четыре расчётные карты ПК ЛИРА:

- `...As1X1.png` — карта `As1X`;
- `...As2X1.png` — карта `As2X`;
- `...As3Y1.png` — карта `As3Y`;
- `...As4Y1.png` — карта `As4Y`.

Рабочая гипотеза для UX: это четыре независимых слоя армирования по направлениям X/Y и граням элемента. Точное соответствие `As1...As4` нижней/верхней арматуре должно быть подтверждено на первом пользовательском прогоне; до этого автоматическая запись обязана явно показывать выбранное назначение каждого слоя.

Файл `АрмированиеПоИзополям_rvt23.rvt` найден как отдельный файл, а не каталог. Без запуска Revit через `BasicFileInfo` подтверждено: формат Revit 2023, язык сохранения Russian, модель workshared/central. Внутренние виды, плиты, отверстия и уже созданная арматура статически не исследовались — они входят в ручную проверку пользователя.

## Текущее состояние

Уже работает на уровне безопасного технического MVP:

- отдельный модуль и кнопка в ленте TrueBIM;
- загрузка готового JSON-контракта зон;
- встроенное offline-распознавание шкалы и цветных hotspot-зон; внешний CLI worker остаётся опциональным override через окружение;
- комплект из четырёх изображений с ролями `As1X`, `As2X`, `As3Y`, `As4Y`, мультивыбором, миниатюрами, ручной коррекцией и проверкой pixel size;
- проверка роли по растровому заголовку ПК ЛИРА с confidence, объяснением источника назначения и блокировкой конфликтов имени/заголовка;
- извлечение `5–8` цветов шкалы на предоставленных картах, фильтрация сетки КЭ, объединение соседних цветных ячеек и построение консервативных convex-hull контуров с confidence;
- ручная таблица проверки результата: исключение зоны, смена класса по легенде и контролируемое объединение зон одного слоя/класса в общий convex hull;
- последовательная обработка четырёх карт с сохранением роли в контурах и диагностике;
- явное назначение верх/низ для каждого расчётного слоя с блокировкой записи при неоднозначности;
- сохранение и повторная загрузка manifest с относительными путями, размерами и SHA-256 исходников;
- preview контуров в окне и управляемые линии на активном 2D-виде;
- выбор одной стены или плиты;
- ручная калибровка координат;
- расчёт и диагностика правил;
- создание по одному пробному стержню на валидную зону с подтверждением;
- логирование и unit-тесты базовых преобразований.

Что пока не позволяет считать плагин производственным:

- числовые границы шкалы (`см²/м`) и сочетания диаметр/шаг распознаются для текущего шаблона ПК ЛИРА; зоны получают диапазон, а карточка показывает подписи его границ;
- узкий классификатор заголовка рассчитан на текущий шаблон экспорта ПК ЛИРА; при другом шрифте или масштабе роль определяется по имени либо вручную;
- легенда с RGB, числовыми диапазонами и подписями сочетаний пока остаётся внутренней managed-моделью и не сериализуется в JSON-контракт `1.0`;
- зоны изображения не сопоставляются с реальным контуром плиты, отверстиями и локальными осями;
- создаются отдельные пробные стержни, а не полноценные наборы арматуры;
- не учитываются защитный слой, грани, нахлёсты, анкеровка, минимальные длины и конструктивная арматура;
- повторный запуск не обновляет ранее созданный результат идемпотентно.

## Целевой пользовательский сценарий

1. Выбрать плиту или набор плит и сохранить локальные направления X/Y.
2. Загрузить четыре карты `As1X`, `As2X`, `As3Y`, `As4Y`; плагин автоматически группирует их в один расчётный комплект и показывает отсутствующие/дублирующиеся слои.
3. Распознать легенду и зоны, проверить наложение на плиту, вручную исправить назначение слоя или границы спорной зоны.
4. Настроить базовое армирование, таблицу допустимых диаметров/шагов, защитный слой и правила усиления; увидеть read-only расчёт и конфликты.
5. Создать армирование только после финального подтверждения, затем получить отчёт: слой, зона, требуемая/принятая площадь, диаметр, шаг, количество и диагностические предупреждения.

## RoadMap

### P0. UX и честные состояния текущего MVP — выполнено в текущем срезе

- Исправить фактический размер rasterized-иконок ленты: `LargeImage` должен быть настоящим 32×32, а не геометрией 16×16 с другим ключом кэша.
- Увеличить общие WPF-токены иконок и централизовать размеры ribbon small/large.
- Перестроить окно в последовательность `источник → зоны → host → правила → создание`.
- Объединить выбор источника и запуск обработки в одном шаге.
- Отключать недоступные действия до выполнения предпосылок и показывать причину в tooltip даже для disabled-кнопок.
- Использовать встроенный offline-runner как рабочий fallback, если внешний CLI worker не настроен; честно показывать ограничения результата.
- Заменить статичный блок «Будущие шаги» живым индикатором готовности `0...4`.
- Спрятать ручную калибровку в раскрываемые дополнительные настройки.
- В UI называть текущую запись «пробным армированием» и отдельно предупреждать, что это не производственная раскладка.

Критерий готовности: новый пользователь понимает следующий шаг без вызова ошибочного `TaskDialog`; опасная кнопка недоступна до валидных правил.

### P1. Контракт комплекта изополей — в работе

- [x] Добавить модель `IsoFieldSourceSet` с четырьмя обязательными ролями: `As1X`, `As2X`, `As3Y`, `As4Y`.
- [x] Поддержать мультивыбор изображений и автоматическое определение роли по имени.
- [x] Добавить ручное переназначение роли, миниатюру каждой карты и проверку одинакового pixel size.
- [x] Обрабатывать четыре карты последовательно и сохранять роль в объединённом результате.
- [x] Дополнить определение роли чтением заголовка внутри изображения; при конфликте с именем файла требовать ручное подтверждение.
- [x] Сохранять manifest комплекта рядом с исходниками, чтобы повторная загрузка была воспроизводимой; проверять размеры и SHA-256.
- [x] Показывать явную таблицу `расчётный слой → X/Y → верх/низ` и блокировать запись до однозначного назначения.
- [ ] После первого пользовательского прогона подтвердить инженерское соответствие `As1...As4` верхней/нижней грани и решить, допустимы ли default-значения.

Критерий готовности: неполный или неоднозначный комплект нельзя отправить на создание; пользователь видит конкретную проблему по каждому файлу.

### P2. Поставляемое распознавание легенды и зон

- [x] Добавить встроенный managed-распознаватель без сети, внешнего процесса и новых native DLL; CLI оставить опциональным override.
- [x] Находить горизонтальную цветовую шкалу отдельно от расчётной области и извлекать её RGB-уровни.
- [x] Фильтровать тонкую сетку КЭ, объединять соседние цветные ячейки, строить convex-hull зоны с confidence и сохранять исходный pixel-space.
- [x] Показывать в окне отдельную цветовую шкалу для каждого слоя и честную диагностику результата.
- [x] Извлекать числовые диапазоны `As` в `см²/м`, проверять монотонность границ и переносить диапазон в подпись зоны.
- [x] Распознавать подписи сочетаний вида `d10s200+d14s200`, проверять полный упорядоченный набор и связывать их с границами диапазонов шкалы.
- [x] Дать пользователю экран коррекции: исключить зону, изменить класс по легенде, объединить выбранные зоны одного слоя/класса; после правки сбрасывать правила и явно обновлять preview.
- Добавить эталонные fixtures из четырёх предоставленных PNG и регрессионные метрики.

Критерий готовности: все четыре эталонные карты обрабатываются офлайн одинаково при повторном запуске; низкий confidence не проходит без ручного подтверждения.

### P3. Геометрическая привязка к плите

- Выбирать конкретную плиту до записи и строить её локальную систему координат.
- Поддержать привязку минимум по двум контрольным точкам и отдельную проверку масштаба.
- Показывать overlay зон поверх контура плиты до создания.
- Обрезать зоны по границе плиты и исключать отверстия.
- Проверять поворот, зеркальность, несовпадение пропорций и выход зон за host.
- Сохранить профиль калибровки для повторного запуска на том же виде/плите.

Критерий готовности: контрольные зоны из PNG попадают в ожидаемые места файла `АрмированиеПоИзополям_rvt23.rvt`, а несовместимая привязка блокирует создание.

### P4. Инженерные правила и производственная раскладка плит

- Зафиксировать единицы исходных значений (`см²/м`) и формулу требуемой площади.
- Добавить базовую сетку и расчёт только дополнительного усиления либо полный подбор — режим выбирается явно.
- Подбирать допустимое сочетание диаметр/шаг с округлением в безопасную сторону.
- Создавать раздельные наборы по направлениям X/Y и верхней/нижней граням.
- Учитывать защитный слой, минимальные отступы от края/отверстий, минимальную длину зоны и конструктивные ограничения.
- Делить/объединять соседние зоны так, чтобы результат был пригоден для выпуска чертежей, а не повторял пиксельный шум.
- Помечать элементы стабильными `TrueBIM`-идентификаторами и исходным zone id.

Критерий готовности: созданная площадь не меньше требуемой, каждый стержень находится внутри допустимой геометрии host, результат разделён на четыре понятных слоя.

### P5. Preview, редактирование и повторный запуск

- Сделать preview будущих наборов без постоянных `DetailCurve` либо с гарантированной очисткой.
- Показать таблицу зон с фильтрами `слой`, `статус`, `диаметр`, `шаг`, `confidence`.
- Разрешить исключить зону, переопределить правило и объединить соседние зоны до записи.
- При повторном запуске сравнивать `добавить / обновить / удалить / без изменений`.
- Обновлять только принадлежащие модулю элементы и не трогать ручную арматуру.
- Поддержать Undo одной транзакцией/группой транзакций.

Критерий готовности: повторный запуск с теми же данными не создаёт дублей; пользователь понимает состав изменений до подтверждения.

### P6. Отчёт, диагностика и контроль качества

- Экспортировать CSV/JSON-отчёт по каждой зоне и итогам слоя.
- Сохранять источник, hash файлов, версию worker, калибровку и профиль правил.
- Отдельно показывать блокирующие ошибки и предупреждения, допустимые по решению пользователя.
- Добавить проверку покрытия зон, пересечений, выхода за host и несоответствия требуемой площади.
- Сформировать короткий итог после создания со ссылкой на лог и отчёт.

Критерий готовности: по результату можно восстановить, из какого комплекта и по каким правилам создан каждый набор.

### P7. Стены, сложная геометрия и выпуск

- После стабилизации плит добавить прямые стены, затем curved/stacked walls.
- Добавить наклонные плиты, сложные контуры и compound structure отдельными проверяемыми срезами.
- Провести smoke на Revit 2023 с предоставленной моделью и регрессию поддерживаемой матрицы TrueBIM.
- Обновить встроенную методичку, manual QA, installer payload и installation report.
- Выпускать production-режим только после подтверждения инженером результатов на эталонной модели.

## Definition of Done для первого производственного релиза

- Четыре карты загружаются как один комплект и имеют однозначное назначение.
- Легенда и зоны воспроизводимо распознаются офлайн, спорные места видны пользователю.
- Overlay совпадает с выбранной плитой и корректно учитывает внешний контур и отверстия.
- Правила явно показывают требуемую и принятую площадь арматуры.
- Создаются полноценные наборы по X/Y и верх/низ, а не одиночные пробные стержни.
- Повторный запуск не создаёт дублей и предварительно показывает diff.
- Все изменения отменяются через Undo, ручная арматура не затрагивается.
- Сохраняется отчёт с источниками, правилами, предупреждениями и созданными element ids.
- Unit/integration-тесты зелёные; ручной smoke пройден на `АрмированиеПоИзополям_rvt23.rvt`.

## Данные, которые нужно подтвердить после первого пользовательского прогона

- точное назначение `As1X`, `As2X`, `As3Y`, `As4Y` относительно верхней/нижней граней;
- создаём полную расчётную арматуру или добавку к заранее заданной базовой сетке;
- разрешённые диаметры, минимальный/максимальный шаг и правило округления;
- требуемые защитные слои, нахлёсты, анкеровка и правила обработки отверстий;
- нужно ли группировать результат в `Area Reinforcement`, `Path Reinforcement`, обычные `Rebar Set` или сохранять несколько режимов.
